# TotTaps Asset Policy

TotTaps should use assets that are safe for commercial web use and friendly for young children.

## Preferred asset sources

Use assets in this order:

1. Original TotTaps artwork, sounds, and UI made specifically for this project.
2. CC0 / public domain assets that allow commercial use without attribution.
3. CC-BY assets only when we can provide clear attribution on a Credits page.
4. Paid royalty-free assets only when the license allows web games and redistribution in the app.

Good starting points to evaluate:

- Kenney assets, prioritizing CC0 packs.
- Pixabay sound effects after checking the asset license page.
- Mixkit game sound effects after checking the Mixkit license terms.
- OpenGameArt only when the individual asset license is CC0 or compatible CC-BY.

Do not assume every asset on a site has the same license. Check the specific download page.

## Avoid

- Brand characters, logos, mascots, or game names from existing franchises.
- Ripped sprites, copied UI, music, sound effects, or screenshots from other games.
- Assets with unclear licenses.
- Assets marked "personal use only" unless TotTaps is never used commercially.
- AI-generated assets that imitate a specific brand, artist, character, or franchise style.

## Approved game concept types

These mechanics are generic enough for TotTaps as long as we create original expression:

- Memory pairs
- Object sorting
- Letter and number tracing
- Bubble or balloon popping
- Hide-and-seek / hidden object
- Pattern sequencing
- Connect the dots
- Tap-in-order paths

## Attribution rules

For every third-party asset, save:

- Source URL
- Author name
- License name
- Download date
- Any attribution text required by the license

If attribution is required, add it to a visible Credits or About section before shipping.

TotTaps has a public Credits page at `credits.html`.

## Folder plan

Use these folders when assets are added:

- `assets/original/` for TotTaps-made assets.
- `assets/cc0/` for public domain assets.
- `assets/credited/` for assets requiring attribution.
- `assets/audio/` for sound effects and music.
